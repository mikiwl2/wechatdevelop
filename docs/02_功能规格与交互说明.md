# 功能规格与交互说明

更新时间：2026-03-08

## 1. 页面结构
1. `pages/ai-quick/ai-quick`：AI试衣主入口。
2. `pages/index/index`：AI衣橱（社区流）。
3. `pages/mine/mine`：我的（登录、统计、作品管理）。
4. `pages/edit-profile/edit-profile`：个人资料编辑。
5. `pages/detail/detail/detail`：作品详情与评论。

## 2. 全局交互规则
1. 自定义 TabBar：三栏 `AI试衣 / AI衣橱 / 我的`。
2. 登录门禁：
- 未登录点击前两栏，强制跳转“我的”并提示“请先登录”。
3. 导航：公共 `navigation-bar` 组件支持返回与首页跳转。

## 3. AI试衣页（ai-quick）
## 3.1 主要按钮
1. 上传个人形象：`choosePersonImage`。
2. 上传衣服图片：`chooseClothesImage`。
3. 一键生成试衣：`onGenerate`。
4. 结果弹窗：发布并保存 / 仅保存 / 不保存。

## 3.2 触发逻辑
1. 生成前校验：两张图片缺一不可。
2. 当前为 mock 生成（2s 延时），结果占位使用人物图。
3. 保存动作：写入 scoped `tryonList` 和 `generateCount`。
4. 发布并保存：`published=true`，可在衣橱展示。
5. 仅保存：只进入“我的试衣”。

## 4. AI衣橱页（index）
## 4.1 主要能力
1. 展示公开作品流（过滤 `published !== false`）。
2. 点赞、收藏即时状态更新。
3. 长按作品：保存到相册 / 分享。
4. 点击卡片进入详情页。

## 4.2 状态处理
1. 互动状态保存在 scoped `indexInteractions`。
2. 列表显示采用 `base + user action` 叠加展示。

## 5. 详情页（detail）
## 5.1 主要能力
1. 作品详情展示（图、作者、时间）。
2. 点赞/收藏切换。
3. 评论输入与发布。

## 5.2 异常处理
1. 若作品不存在，显示占位并提示“作品不存在或已删除”。
2. 评论空文本不可发送。

## 6. 我的页（mine）
## 6.1 未登录态
1. 仅展示登录/注册入口。
2. 不展示 AI 功能内容。

## 6.2 已登录态
1. 顶部用户信息与统计（保存作品/生成次数/点赞）。
2. Tabs：我的试衣 / 我的点赞 / 我的收藏。
3. 作品操作：预览、长按操作、删除、批量删除。
4. 悬浮球：仅保留“+ 生成更多”。

## 6.3 登录/退出逻辑
1. 登录：
- 首次可选择是否采用微信头像昵称。
- 后续登录不重复弹窗。
2. 退出：
- 清空登录态（userInfo），保留同账号 scoped 历史数据。
- 清空 `globalData` 的临时作品计数，防止内存串数据。

## 7. 个人资料页（edit-profile）
1. 可编辑头像、昵称、性别、身高体重、三围、体型、风格偏好等。
2. 保存后写 scoped `profile` 与 `userInfo`。
3. 保存后返回“我的”应可看到昵称头像更新。

## 8. 当前已知限制
1. AI 生成仍为 mock，不是线上模型结果。
2. 数据仅本地存储，未实现服务端账号同步。
3. 分享为小程序原生菜单能力，未做素材加工与分享卡配置。
