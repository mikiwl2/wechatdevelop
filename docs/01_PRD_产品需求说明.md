# PRD 产品需求说明

更新时间：2026-03-08
版本：`v0.9.0`

## 1. 产品定位
一款微信小程序端 AI 试衣产品，面向普通用户提供：
1. 快速上传人物与服装图片。
2. 生成试衣结果并沉淀为作品。
3. 在衣橱社区浏览、点赞、收藏、评论。
4. 在“我的”页管理作品和个人资料。

## 2. 目标与指标
## 2.1 业务目标
1. 完成最小可用闭环（登录、创作、沉淀、互动）。
2. 降低首次使用门槛，保障登录后数据可持续。

## 2.2 核心指标（建议）
1. `登录转化率`：进入“我的”后完成登录比例。
2. `首图生成率`：登录后完成一次生成比例。
3. `作品留存率`：7日内仍有作品浏览/互动行为。
4. `互动率`：点赞/收藏/评论行为占浏览行为比例。

## 3. 用户与场景
1. 新用户：首次进入，仅可看登录入口，登录后完成第一次试衣。
2. 老用户：登录后继续查看历史作品并继续创作。
3. 浏览用户：在 AI 衣橱查看公开作品并互动。

## 4. 范围定义
## 4.1 本期范围（In Scope）
1. 登录门禁与账号状态管理。
2. AI试衣流程（上传、生成、保存、发布）。
3. AI衣橱流（列表、点赞、收藏、长按保存/分享、详情跳转）。
4. 我的页（统计、作品管理、批量删除、资料入口）。
5. 个人资料页（昵称头像+体型资料编辑保存）。

## 4.2 非本期范围（Out of Scope）
1. 真实 AI 推理服务接入与计费体系。
2. 后端用户中心与跨设备同步。
3. 内容审核、风控、举报系统。
4. 商业化模块（订阅、会员、广告）。

## 5. 功能需求（高层）
1. 登录与门禁：未登录不可使用 AI试衣、AI衣橱、详情互动能力。
2. 作品生成：需同时上传人物图和服装图，生成结果可保存/发布。
3. 作品管理：我的页支持查看、删除、批量删除、预览。
4. 互动能力：衣橱/详情支持点赞、收藏、评论，状态可回显。
5. 资料管理：用户可编辑头像昵称与身材资料，保存后应反映到“我的”。

## 6. 非功能需求
1. 稳定性：退出登录后不串号、不串数据。
2. 一致性：同一账号再次登录可恢复已有数据。
3. 兼容性：微信基础库 `3.14.3` 下可运行；自定义 tab 与导航可正常展示。
4. 可维护性：存储和门禁逻辑集中化，减少分散重复实现。

## 7. 验收标准（发布门槛）
1. 未登录状态下，点击“AI试衣/AI衣橱”均跳转“我的”登录入口。
2. 首次登录可选择是否使用微信头像昵称；后续登录不重复弹窗。
3. 同一微信号退出再登录可恢复资料与作品。
4. 不同账号登录后互不串数据。
5. 作品生成、保存、发布、查看、删除链路完整可用。
