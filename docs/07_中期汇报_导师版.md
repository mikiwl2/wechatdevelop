## 3. 已实现功能（可演示）

### 3.1 账号与登录体系

- 未登录状态下，只允许停留在“我的”页进行登录。
- 登录后可访问 AI试衣、AI衣橱、我的完整功能。
- 退出登录后回到未登录视图，受保护页面不可继续使用。
- 基于 `scoped storage` 实现账号数据隔离，避免账号间串数据。

### 3.2 AI试衣流程

- 支持上传人物图与服装图。
- 模拟生成结果后可二选一：
  - 仅保存到“我的试衣”
  - 发布并保存（进入公共展示流）
- 生成次数统计联动“我的”页展示。

### 3.3 AI衣橱（作品流）

- 展示公开发布作品，支持点赞、收藏、长按操作（保存/分享）。
- 支持从衣橱进入详情页查看作品内容与互动状态。
- 已修复“不同账号登录看不到其他用户已发布作品”的问题：
  - 引入公共作品池 `publicPublishedWorks`
  - 登录账号间可共享查看公共发布作品。

### 3.4 我的页面

- 个人头图、头像昵称、作品统计、分栏（我的试衣/我的点赞/我的收藏）。
- 试衣作品支持长按删除、批量删除模式。
- 资料编辑页支持昵称头像修改并回写主页显示。
- 已按需求移除“设置”按钮，减少无效入口。

### 3.5 详情页互动

- 支持点赞、收藏、评论发布、图片预览。
- 详情页已兼容公共作品来源，避免“从衣橱点开提示作品不存在”。

## 4. 关键问题与修复记录（阶段重点）

1. 修复 `mine.wxss` 编译错误（BOM/编码导致 WXSS 首字符异常）。
2. 修复登录门禁缺失：退出后仍可访问受保护功能的问题。
3. 修复重新登录后作品不即时刷新的问题（跨页切换后才显示）。
4. 优化 `wx.getUserProfile` 调用频率，避免频繁触发告警。
5. 修复同一微信反复登录生成新空账号的问题，提升账号稳定性。
6. 修复资料页已保存但 mine 页未同步显示的问题。
7. 修复公共发布流跨账号不可见问题（当前已支持查看他人发布作品）。

## 5. 目前存在的边界与说明

- 当前为前端演示阶段，AI生成与社区数据主要基于本地存储模拟。
- 作品“公共可见”范围目前是同设备本地公共池，不等同于云端全网分发。
- 若进入毕业设计后期，需接入后端账号体系与云存储实现真正跨设备同步。

## 6. 测试与验收方式

本阶段以手工回归为主，围绕以下主链路进行验证：

1. 未登录拦截 -> 登录 -> 完整功能可用。
2. 试衣生成 -> 保存/发布 -> 我的页与衣橱页联动。
3. 退出登录 -> 再登录 -> 同账号数据恢复。
4. 不同账号切换 -> 数据隔离正确 -> 公共发布可见。
5. 资料修改 -> mine 页即时更新。
6. 删除作品 -> 个人列表与公共列表状态一致。
