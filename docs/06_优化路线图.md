# 优化路线图（Roadmap）

更新时间：2026-03-08

## 1. P0（下个版本建议优先）
1. 接入真实 AI 生成服务（替换前端 mock 逻辑）。
2. 引入服务端账号体系（openid/session + 云端作品与资料）。
3. 建立作品主键与后端 ID 对齐，避免纯本地时间戳冲突。

## 2. P1（稳定性与增长）
1. 互动数据服务端化，支持跨设备同步。
2. 分享能力增强：自定义分享卡、落地页、拉新追踪。
3. 资料完善引导（新手引导、资料完整度进度条）。
4. 生成流程优化：失败重试、进度反馈、队列状态。

## 3. P2（体验提升）
1. 图标体系统一（从 emoji 逐步迁移到统一 icon 资源）。
2. UI 细节打磨（状态页、空态插画、动效节奏一致性）。
3. 可观测性：埋点、错误上报、漏斗分析仪表盘。

## 4. 管理建议
1. 以两周为迭代周期，按 `P0 -> P1 -> P2` 逐步交付。
2. 每个迭代同步更新：PRD、变更记录、测试清单。
3. 所有线上问题必须关联到版本记录和回归用例。

## 5. 目标版本建议
1. `v1.0`：真实 AI + 服务端存储 + 基础运营闭环。
2. `v1.1`：互动与分享增长能力完善。
3. `v1.2`：体验打磨与数据化运营。
