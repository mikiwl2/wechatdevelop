# 技术方案与数据设计

更新时间：2026-03-08

## 1. 架构概览
1. 前端框架：微信小程序原生。
2. UI：自定义 TabBar + 自定义 navigation-bar。
3. 渲染配置：`app.json` 使用 Skyline 配置（需与开发工具设置一致）。

## 2. 状态与存储
## 2.1 全局态（app.globalData）
1. `userInfo`
2. `tryonList`
3. `generateCount`
4. `userKey`

## 2.2 存储隔离策略（scoped）
1. 作用：不同账号数据隔离，避免串号。
2. 规则：`{baseKey}__{userKey}`。
3. 统一工具：`utils/scoped-storage.js`。
4. 关键键：
- `userInfo`
- `profile`
- `tryonList`
- `generateCount`
- `indexInteractions`
- `commentsById`

## 2.3 账号映射
1. 映射表：`wxAccountMap`。
2. 指纹：`nickName|avatarUrl`。
3. 生成 userKey：hash + 时间戳。

## 3. 鉴权门禁
1. 统一守卫：`utils/auth-guard.js`。
2. 页面通过 `requireLoginGate` 进行拦截与跳转。
3. 判断顺序：
- app.isLoggedIn
- scoped userInfo
- scoped fallback（activeUserKey 对应 userInfo）

## 4. 核心数据模型
## 4.1 作品对象
1. `id`
2. `image`
3. `avatar`
4. `nickname`
5. `likes`
6. `saves`
7. `createdAt`
8. `title`
9. `published`

## 4.2 互动对象
1. `indexInteractions[itemId].liked`
2. `indexInteractions[itemId].saved`

## 4.3 评论对象
1. `id`
2. `nickname`
3. `avatar`
4. `text`
5. `createdAt`

## 5. 兼容与配置注意事项
1. `project.private.config.json` 的 `skylineRenderEnable` 应与 `app.json` 渲染策略统一。
2. 文档与代码文件统一 `UTF-8 (No BOM)`，避免 wxss/wxml 编译异常。

## 6. 技术债（后续）
1. 目前为本地存储方案，跨设备与服务端同步能力缺失。
2. userKey 依赖微信资料指纹，存在边界条件（昵称/头像变化）。
3. 缺少统一 service 层与类型约束（建议后续补齐）。
