## 4. 云函数功能总表

| 云函数 | 作用 | 关键 action |
|---|---|---|
| `auth_v2` | 登录与资料主服务 | `login`(默认), `syncProfile`, `getProfile`, `saveProfile` |
| `images_v1` | 头像服务（单用户单当前头像） | `saveAvatar`, `setCurrentAvatar`, `deleteAvatar`, `listAvatars`, `getCurrentAvatar`(默认) |
| `works_v1` | 试衣作品服务 | `createWork/saveWork/upsertWork`, `listMine`, `listPublic`, `listByIds`, `getWork`, `setPublish`, `deleteWork` |
| `works_social_v1` | 社交互动服务 | `toggleLike`, `toggleSave`, `getInteraction`, `batchGetInteractions`, `listMyInteractions`, `addComment`, `listComments` |
| `auth` | 旧版登录函数（历史保留） | `login`(默认), `syncProfile` |

## 4.1 `auth_v2` 细节

1. 以 `OPENID` 作为 `userKey`。
2. 首登自动建 `users` 文档（默认昵称 `wx_user`）。
3. 支持重复脏数据自愈：
   - 根据 `userKey/_openid` 查询候选文档
   - 选“最可信”文档作为主文档
   - 清理空白引导文档
4. `saveProfile` 会同步更新 `users.nickname/avatarUrl/profile`。

## 4.2 `images_v1` 细节

1. `images` 集合采用**确定性文档 ID**（同用户固定 `img_xxx`）。
2. `saveAvatar` 行为：
   - 覆盖写当前头像文档；
   - 删除同用户历史头像文档与历史文件；
   - 同步 `users.avatarUrl` 和 `users.profile.avatarUrl`。
3. 结果：同一用户只保留“当前头像”一份，便于管理。

## 4.3 `works_v1` 细节

1. `createWork` 支持幂等 upsert（同 `workId` 可更新）。
2. 写入时校验 `resultFileId/resultUrl` 必填。
3. 计数安全：
   - 新建作品强制 `likes/saves/comments = 0`；
   - 不信任前端传入计数字段（防篡改）。
4. 权限：
   - 未发布作品仅作者可读。
5. 删除：
   - 支持软删/硬删；
   - 可选 `purgeFiles=true` 删除云存储文件；
   - 自动清理该作品的互动和评论数据。
6. `updateStats` 已禁用，计数统一交由 `works_social_v1`。

## 4.4 `works_social_v1` 细节

1. 互动文档使用确定性 `_id`：`md5(userKey + workId)`。
2. 点赞/收藏切换后会实时重算作品计数并回写 `works`。
3. 若同用户对同作品 `liked=false && saved=false`，互动文档会删除（减存储噪音）。
4. 评论新增后会同步更新作品 `comments` 计数。
5. `listMyInteractions(type=all)` 支持去重分页统计逻辑。
